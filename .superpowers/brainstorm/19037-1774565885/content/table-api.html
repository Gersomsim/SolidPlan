API de la tabla

¿Cómo se define qué columnas mostrar y cómo se renderizan las celdas?

A

Config object array

Columnas definidas como array de objetos. Rápido para tablas simples.

columns: TableColumn[] = [
  { key: 'code',   label: 'Código',  sortable: true },
  { key: 'name',   label: 'Nombre',  sortable: true },
  { key: 'status', label: 'Estado',  type: 'badge'  },
  { key: 'budget', label: 'Presup.', type: 'currency' },
];

<lib-table
  [columns]="columns"
  [data]="projects"
  [loading]="loading"
  (rowClick)="onRowClick($event)"
/>

✓ Pros

  • Muy simple para tablas estándar
  • Poca configuración

✗ Contras

  • Limitado para celdas custom (badges, acciones)
  • Necesita tipos especiales por cada caso
B

Columnas declarativas con ng-template

Cada columna se declara en el template. Máximo control sobre el render de cada celda.

<lib-table [data]="projects" [loading]="loading">

  <lib-column key="code" label="Código" sortable />

  <lib-column key="status" label="Estado">
    <ng-template libCell let-row>
      <lib-badge [status]="row.status" />
    </ng-template>
  </lib-column>

  <lib-column label="Acciones" align="right">
    <ng-template libCell let-row>
      <lib-button size="sm" (click)="edit(row)">Editar</lib-button>
    </ng-template>
  </lib-column>

</lib-table>

✓ Pros

  • Render custom por celda (badges, botones, iconos)
  • Patrón familiar (Angular Material, PrimeNG)
  • Flexible sin límites

✗ Contras

  • Más verboso en el template
C

Híbrido (recomendado)

Config object para columnas simples + slot ng-template opcional por columna para celdas custom.

columns: TableColumn[] = [
  { key: 'code',   label: 'Código',  sortable: true },
  { key: 'name',   label: 'Nombre',  sortable: true },
  { key: 'status', label: 'Estado'  },
  { key: 'actions', label: '', align: 'right' },
];

<lib-table [columns]="columns" [data]="projects"
           (sortChange)="onSort($event)">

  <!-- Solo defines templates para las celdas que necesitan custom render -->
  <ng-template libCell="status" let-row>
    <lib-badge [status]="row.status" />
  </ng-template>

  <ng-template libCell="actions" let-row>
    <lib-button size="sm" variant="ghost" (click)="edit(row)">
      Editar
    </lib-button>
  </ng-template>

</lib-table>

✓ Pros

  • Simple por defecto, flexible cuando se necesita
  • Columnas simples sin template extra
  • Custom render solo donde importa

✗ Contras

  • Dos patrones (config + template)
